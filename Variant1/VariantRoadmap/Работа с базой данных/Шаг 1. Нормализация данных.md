Файлы для импорта хранятся в соответствующей папке материалом экзамена. Примерная структура выглядит как на скриншоте ниже. Файлы для импорта даны в различных форматах, чтобы экзаменуемый собрал их в едином удобном виде. 
![](./assets/Pasted%20image%2020260524233252.png)

Для начала следует просмотреть все выданные файлы и создать по описанию моделей (столбцов), в новом excel файле, соответствующие страницы (листы).
В некоторых файлах будет изменена кодировка. Чтобы ее поменять следует перейти в новый файл -> Данные -> из текстового/CSV 
![](assets/Pasted%20image%2020260524235516.png|535)

![](assets/Pasted%20image%2020260524235953.png)

После этих действий нужно выбрать "испорченный файл" и указать правильную кодировку (UTF-8). Теперь данные отображаются корректно. Желательно вставлять данные в новый файл, в котором далее будет происходить нормализация данных.
![](assets/Pasted%20image%2020260525000114.png)

В итоговом файле для удобства лучше сразу переименовывать листы. 
![](assets/Pasted%20image%2020260525000333.png)

Далее в этот же файл из следующего текстового документа можем добавить "клиентов". Просто выделяем все значения и вставляем в самую первую ячейку.
![](assets/Pasted%20image%2020260525000915.png)

Для быстрого анализа и поиска дубликатов переходим на Главную страницу -> условное форматирование -> правила выделения ячеек -> повторяющиеся значения
![](assets/Pasted%20image%2020260525001041.png)

В следующем файле для импорта мы видим то что данные для таблицы "многие-ко-многим" поскольку имя файла и его содержимое говорит о том, что у нас есть пересечение между услугами и клиентами (у одного клиента может быть много услуг и у одной услуги может быть много клиентов)
![](assets/Pasted%20image%2020260525001340.png)

Также в этом же файле с клиентами и услугами на следующей странице можно обнаружить описание для таблицы с услугами (исходя из наименования листа и названий таблицы). Эти данные помогут для правильного формата таблицы в базе и правильного порядка столбцов в файле импорта (также важно увидеть, то что временные данные передаются в формате секунд по описанию одного из столбца).
![](assets/Pasted%20image%2020260525001429.png)

После того, как мы перенесли в единый файл все данные из файлов импорта, нужно перейти к самой нормализации данных. 

Основные моменты на которые следует обратить внимание в первую очередь:
- Данные дублируются
- Данные в ненормированном формате
	- у некоторых записей есть пробелы
	- у некоторых записей есть лишние символы
	- у некоторых записей неверная дата-время
	- у числовых записей присутствуют символы (и наоборот)

Для нормализации данных для начала следует убрать все дубли (Выдели таблицу → Данные → Удалить дубликаты)

Далее можно нормализовать номера телефонов и даты в таблице клиентов. Как видно на скриншотах данные имеют различный вид. Проще всего для номеров привести к виду 7999999999, а для дат выбрать ДД-ММ-ГГ. 
![](assets/Pasted%20image%2020260525002553.png)

![](assets/Pasted%20image%2020260525002441.png)

Для номеров телефона можно воспользоваться функцией "найти и заменить" (главная -> редактирование -> найти -> заменить)
![](assets/Pasted%20image%2020260525003022.png)

В открывшемся поле заменяем пробелы, скобки, запятые и прочие неподобающие символы на пустой знак (ничего).
![](assets/Pasted%20image%2020260525003115.png)

Получаем телефоны следующего формата (количество символов разное и его менять нельзя - это независимый от нас фактор предусмотренный экзаменаторами).
![](assets/Pasted%20image%2020260525003245.png)

Даты приводим к единому значению ДД-ММ-ГГ через точку. Выделяем столбик с датой, затем переходим: Главная -> число -> выпадающий список -> краткий формат даты
![](assets/Pasted%20image%2020260525003438.png)

Также после первичных изменений просматривает все ячейки на предмет "странностей" (пробелы, левые символы, табуляция и тд). Изменяем таким образом все таблицы (на всех листах) в файле.

Также нормализацию стоит выполнить над полем скидки. В данном случае лучше выделить скидку как обычное число - убрать текст (там где нет - скидка 0) и убрать символы. 
![](assets/Pasted%20image%2020260525004938.png)

В целом все изменения над таблицами и колонками по большей части делаются через "Найти и заменить".

Далее сложный момент. В одном из столбцов есть и минуты и секунды в формате "123 мин.". Чтобы привести все к секундам я скопировал столбик в отдельную колонку, затем заменил через выделение " мин." на пустое значение. Далее к этому в соседнем столбике я прогнал формулу "=ЕСЛИОШИБКА(F3*60;ЗНАЧЕН(ПОДСТАВИТЬ(F3;" сек.";"")))"  и затем растянул ее. Таким образом я получил значения секунд в последнем столбце.
![](assets/Pasted%20image%2020260525011155.png)

Также существует формат даты-времени. Его можно посмотреть и поменять через "дополнительные форматы"
![](assets/Pasted%20image%2020260525011458.png)

Также поскольку при создании базы данных (а она у нас реляционная) у нас в таблице многие-ко-многим не будет поля "name" , а будет индекс (ссылка) к таблице клиентов. Поэтому в этой таблице следует заменить имена клиентов на индексы (ID)
![](assets/Pasted%20image%2020260525011613.png)

Проставляем обычным перечислением в таблице с клиентами ID
![](assets/Pasted%20image%2020260525012356.png)

Важно, чтобы данные были уникальными. Переходим во вкладку данных -> работа с данными -> удалить дубликаты
![](assets/Pasted%20image%2020260525012632.png)

Скопировав колонку с клиентами и их ID на страницу с клиентами, а затем переместив ID в правую часть - пишем в новой клетке такую формулу и растягиваем ее. Получаем ID. Заменяем его в поле с клиентами
![](assets/Pasted%20image%2020260525013426.png)

Тоже самое проделываем и с услугами и получаем таблицу такого формата.
![](assets/Pasted%20image%2020260525013850.png)

Важно выставить верный индекс (НЕ ОТ НАЧАЛА СТРОКИ, А ОТ НАЧАЛА ПЕРЕЧИСЛЕНИЯ УСЛУГ). Из-за этой ошибки у меня индексы стали на -1. потому что 1 было - "наименование".

Далее последнее изменение в этом файле - синхронизация заголовков. Я меняю положения заголовков у услуг, поскольку в файле это указано. В этих заголовках я вижу "Описание" для услуги, которого у меня нет в действительности

В качестве описания я просто добавлю слово "услуга" для каждого столбца
