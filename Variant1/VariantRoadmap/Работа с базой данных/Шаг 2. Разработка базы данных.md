После нормализации данных для импорта следует создать саму базу данных. 

Для разработки будет использоваться приложение DBeaver Community вместе с PostgreSQL. 

Создавать базу данных можно через UI приложения, но также можно создать через SQL-скрипт. Моя текущая база данных будет на localhost с базовыми настройками пользователя. Настройки подключения далее нам понадобятся при разработке и миграциях. 

![](../assets/Pasted%20image%2020260527180534.png)

После создания скрипта, нужно определить, какие вообще сущности и поля нужно создать. 

Исходя из предыдущего шага, можно понять, что нам нужно создать 5 таблиц: Услуги, гендер, клиенты, услуги_клиенты, изображения. 

Сначала нужно начать с таблиц, с наименьшими связями. То есть - с изображений. Затем гендер, клиенты, а после услуги. В самый последний момент создается таблица многие-ко-многим. Таблицы создаются с полями, перечисленными в эксель-файле. 

Создание таблицы изображений. Serial отвечает за автоматическую инкрементацию ID. 

``` SQL
create table images (
id serial primary key,
main_image varchar(100) not null
);
```

Создание таблицы гендеров

``` SQL
create table genders (
id serial primary key,
name varchar(10) not null
);
```

Создание клиентов. В самом конце у нас должна быть ссылка на таблицу гендеров. Именно поэтому прежде чем создавать клиентов, мы создали таблицу с гендерами. Это связь 1 ко многим, потому что 1 гендер может быть у нескольких клиентов, но 1 клиент имеет 1 гендер

``` SQL
create table clients (
id serial primary key,
last_name varchar(100) not null,
first_name varchar(100) not null,
patronymic varchar(100),
birth_date DATE,
register_date DATE,
email varchar(100) not null,
phone varchar(15) not null,
gender_id int references genders(id)
);
```

Похожую таблицу создаем для услуг

``` SQL
create table services (
id serial primary key,
name varchar(200) not null,
cost int not null,
duration_sec int not null,
description varchar(100) not null,
discount int,
image_id int references images(id)
);
```

Последняя таблица - многие ко многим. Тут уже используются вместо DATE - TIMESTAMP. Потому что у нас указано помимо даты, еще и конкретное время.

``` SQL 
create table services_clients (
id serial primary key,
service_id int references services(id),
client_id int references clients(id),
start_provide_service timestamp default current_timestamp
);
```
